Detailed Description

Device driver core.

The device driver core is responsible for internal handling of device drivers. Registering, and unregistering them for example.

void dev_sync_lock ( struct device *  dev,
unsigned  ms 
)

Lock a device with synchronization.

Parameters
devDevice to lock.
msMiliseconds of synchronization required.

Some devices need a certain amount of miliseconds to process information internally after an operation. Code surrounded by dev_sync_lock() and dev_sync_unlock() cannot be accessed for ms miliseconds after dev_sync_unlock() has been called.
